4.1.0

Paste to Upload

Before some of the latest HTML5 features came out, in order to upload an image from another web page, you must save the image to your local file system and then either drop it into an uploader instance or select it from the “choose files” dialog. Certainly copying and pasting is a much more efficient way to do this. Note that you can also use this feature to upload any image currently present in your clipboard. For example, suppose you take a screenshot and add it directly to your clipboard. You can then paste it into your Fine Uploader instance.

Note:

This feature is only supported by browsers that feature the Clipboard API. Currently, only Chrome & Opera 15+ support this completely.

Enable this feature by providing any HTMLElement here.

paste option (UI mode)

Note that all core mode options are also available.

Option Name

Type

Default

Description

promptForName

Boolean

false

If set to true, the showPrompt function will be called in an attempt to ask the user to provide a name for the associated pasted image. See the styling documentation for more details on overriding showPrompt.

namePromptMessage

String

‘Please name this image’

Text that will appear in the showPrompt dialog, asking the user to name the pasted image. Note that an appropriate extension will always been appended to the name supplied by the user automatically. If you override the pasteReceived event handler AND set this option to true, your callback will be ignored.

Events

Event

Parameters

Description

pasteReceived

Blob blob

Called when a pasted image has been received, but before it is uploaded. You may return false to prevent the paste from being processed. You may also return a qq.Promise.
