4.0.0

Options S3

If you are using the S3 uploader, all of Fine Uploader's options are still present, and most of them still function the same way. However, some of the options are slightly different. This page will list any new or different options for the S3 uploader. Any option not listed here can be assumed to have the same behavior for the Traditonal or S3 uploader.

Core Mode

chunking option

Option Name

Type

Default

Description

partSize

Number

5242880

The maximum size of each part, in bytes.

Note

The chunking.paramNames option does not apply to S3.

cors option

Option Name

Type

Default

Description

allowXdr

Boolean

true

Enables or disables cross-domain ajax calls (if expected property is true) in IE9 and older.

iframeSupport option

Option Name

Type

Default

Description

localBlankPagePath

String

null

This is required if you plan on supporting browsers that do not implement the File API, such as IE9 and older. This must point to a blank page on the same origin/domain as the page hosting Fine Uploader.

objectProperties option

Option Name

Type

Default

Description

acl

String

'private'

This value corresponds to a canned ACL

key

String or Function

'uuid'

Describes the object key used to identify the file in your S3 bucket. Possible values are 'uuid', 'filename' or a function. If the value is a function, Fine Uploader S3 will pass the associated file ID as a parameter when invoking your function.

reducedRedundancy

Boolean

false

Set this to true if you would like to use the reduced redundancy storage class for all objects uploaded to S3.

resume option

Option Name

Type

Default

Description

recordsExpireIn

Integer

7

Number of days before resumable file records expire.

Note

The resume.paramNames option does not apply to S3.

request option

Option Name

Type

Default

Description

endpoint

String

null

URL for your S3 bucket. All valid bucket URLs documented by Amazon are supported, including custom domains. SSL is also supported.

accessKey

String

null

Your AWS public key. NOT YOUR SECRET KEY.

params

Object

{}

Parameters passed along with each upload request.

filenameParam

String

'qqfilename'

Part of the parameter name that contains the name of the associated file which may differ from the key name. Prefixed with 'x-amz-meta-' by Fine Uploader.

signature option

Option Name

Type

Default

Description

endpoint

String

null

The endpoint that Fine Uploader can use to send policy documents (HTML form uploads) or other strings to sign (REST requests) before sending requests off to S3.

uploadSuccess option

Option Name

Type

Default

Description

endpoint

String

null

An endpoint that Fine Uploader should POST to when a file has been succcessfully uploaded to S3. The parameters include the key, name, UUID, and bucket associated with the file.

params

Object

{}

Any additional parameters to attach to upload success file requests. Note that Fine Uploader will still send the bucket, key, filename, and UUID as well
